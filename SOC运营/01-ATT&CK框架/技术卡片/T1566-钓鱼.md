# T1566.001 - 鱼叉式钓鱼附件

## 基本信息
- 父技术：[T1566-钓鱼](T1566-钓鱼.md)
- 全称：Spearphishing Attachment
- 中文：鱼叉式钓鱼附件
## 技术描述
攻击者发送带有恶意附件的电子邮件，诱导目标打开附件执行恶意代码。常见附件类型：
- 办公文档（含宏）：.docm, .xlsm
- 可执行文件（伪装）：.exe伪装成.pdf.exe
- 压缩包（绕过检测）：.zip, .iso, .img
- 快捷方式（无文件）：.lnk指向恶意脚本
## 检测思路

### 1. 邮件层面
| 检测点 | 数据源 | 具体方法 |
|-------|--------|---------|
| 附件类型异常 | 邮件网关日志 | 检测.exe/.zip/.iso等高风险扩展名 |
| 发件人伪造 | 邮件日志 | SPF/DKIM/DMARC验证失败 |
| 首次发件人 | 邮件日志 | 历史数据库匹配，新域名告警 |
### 2. 终端层面
| 检测点 | 数据源 | 具体方法 |
|-------|--------|---------|
| Office宏执行 | Sysmon EID1 | winword.exe启动powershell/cmd |
| 异常子进程 | EDR进程监控 | 文档编辑器启动系统程序 |
| 文件释放 | Sysmon EID11 | 临时目录出现可执行文件 |
### 3. 网络层面
| 检测点      | 数据源   | 具体方法        |
| -------- | ----- | ----------- |
| 打开附件后的外联 | 防火墙日志 | 文档进程首次外联    |
| 可疑域名解析   | DNS日志 | 新注册域名、DGA特征 |
## 检测规则（Sigma格式）

```
title: 钓鱼附件执行检测-Office宏启动系统程序
status: experimental
description: 检测Office应用程序启动可疑子进程，可能是宏病毒或钓鱼附件执行
logsource:
    category: process_creation
    product: windows
detection:
    selection_parent:
        ParentImage|endswith:
            - '\winword.exe'
            - '\excel.exe'
            - '\powerpnt.exe'
            - '\mspub.exe'
    selection_child:
        Image|endswith:
            - '\cmd.exe'
            - '\powershell.exe'
            - '\pwsh.exe'
            - '\wscript.exe'
            - '\cscript.exe'
            - '\mshta.exe'
            - '\rundll32.exe'
            - '\regsvr32.exe'
    condition: selection_parent and selection_child
falsepositives:
    - 正常的自动化办公流程（需白名单）
level: high
tags:
    - attack.initial_access
    - attack.t1566.001
    - attack.execution
    - attack.t1059
```
 
# T1566.002 - 鱼叉式钓鱼链接
## 基本信息
- 父技术：T1566钓鱼
- 全称：Spearphishing Link
- 中文：钓鱼-链接
## 技术描述
攻击者发送包含恶意URL的电子邮件，诱导目标点击恶意链接。常见攻击手法：
- **凭证收集**：链接指向伪造的登录页面（如仿造Office 365、企业邮箱），骗取账号密码。
- **恶意下载**：链接指向恶意文件（.exe, .docm, .zip）的下载地址。
- **漏洞利用**：链接指向包含浏览器漏洞利用代码的页面。
- **重定向链**：链接经过多次302跳转，最终到达恶意站点，以此绕过安全网关检测。
## 检测思路
### 1. 邮件层面
| 检测点     | 数据源    | 具体方法                                                           |
| ------- | ------ | -------------------------------------------------------------- |
| url检测   | 邮件网关日志 | 检测正文中的纯IP地址、端口号访问、新注册域名                                        |
| 链接与正文不符 | 邮件内容分析 | 检测 `<a>` 标签的显示文本与实际跳转URL不一致（如显示"Microsoft"但指向"ms-support.xyz"） |
| 短链接滥用   | 邮件网关   | 识别 bit.ly, tinyurl 等短链接服务，需进一步还原                               |
### 2. 终端层面
| 检测点    | 数据源        | 具体方法                                        |
| ------ | ---------- | ------------------------------------------- |
| 浏览器进程链 | EDR/Sysmon | 检测浏览器 点击链接后，启动了其他可疑进程（如Office程序启动，意味着下载了文档） |
| 凭证输入行为 | DLP/EDR    | 检测用户在非标准HTTPS域名下进行了键盘大量输入（需高级行为分析）          |
### 3. 网络层面
| 检测点          | 数据源     | 具体方法                               |
| ------------ | ------- | ---------------------------------- |
| 首次访问         | DNS日志   | 检测内网主机首次请求从未访问过的域名                 |
| 快速通量         | DNS日志   | 检测域名解析IP频繁变动，或生存时间极短，疑似C2或钓鱼站点     |
| 异常User-Agent | Web代理日志 | 检测非浏览器UA访问网页，或过时的浏览器版本（可能为自动化扫描工具） |
| 文件下载         | 代理日志    | 检测点击链接后短时间内发起了可执行文件的下载请求           |
## 检测规则（Sigma格式）

``` yaml
title: 可疑的浏览器子进程启动-疑似钓鱼链接下载
status: experimental
description: 检测浏览器访问链接后，启动了高风险子进程（如Office程序、脚本解释器），可能通过钓鱼链接下载并执行了恶意文件。
logsource:    
  category: process_creation    
  product: windows
detection:    
    selection_browser:        
       ParentImage|endswith:            
        - '\chrome.exe'            
        - '\firefox.exe'           
        - '\msedge.exe'           
        - '\iexplore.exe'    
   selection_suspicious_child:        
     Image|endswith:            
        - '\winword.exe'   # 下载了Word文档并自动打开            
        - '\excel.exe'            
        - '\powerpnt.exe'            
        - '\cmd.exe'       # 浏览器直接启动CMD（罕见）            
        - '\powershell.exe'            
        - '\mshta.exe'    
condition: selection_browser and selection_suspicious_childfalsepositives:    
    - 用户手动下载并打开了正常的Office文档    
    - 浏览器插件或扩展程序的正常更新行为
level: medium
tags:
    - attack.initial_access    
    - attack.t1566.002    
    - attack.execution
```
