# 描述

内置系统监视器 (Sysmon) 是 Windows 11 和 Windows Server 2025 上的[可选 Windows 功能](https://learn.microsoft.com/zh-cn/windows/client-management/client-tools/add-remove-hide-features?tabs=gpo&pivots=windows-11)，启用此功能后，在系统重新启动后仍保持驻留状态，以监视系统活动并将其记录到 Windows 事件日志。
简单来说，**Sysmon更像是一个极度专注的“数据记录仪”**，专注于**深度日志记录**的免费工具。
# 和EDR对比
即便最先进的EDR，出于性能或架构考虑，也无法记录所有系统活动。Sysmon则像一个高保真的“黑匣子”，提供了许多Windows自带日志或EDR默认不记录的“额外”信息。
- **进程树与父进程**：EDR可能记录进程创建，但Sysmon能清晰呈现完整的父子进程关系，这对溯源攻击至关重要。
- **网络连接的原始信息**：Sysmon能记录下发起连接的**进程名、PID、IP、端口**，甚至传输的数据哈希，信息粒度远超常规日志。
- **更隐蔽的持久化手段**：Sysmon能监控计划任务、WMI事件订阅等高级持久化机制，这些是很多EDR的“盲点”。
Sysmon提供了一个免费、中立的“数据底座”，让企业可以：
- **建立双检测机制**：将Sysmon日志也接入SIEM，对照EDR告警，发现被遗漏的威胁。
- **降低核心成本**：对于预算有限的组织，Sysmon + 开源SIEM（如ELK）是构建自有安全监控体系的高性价比方案。
### 部署Sysmon的考量：需要安装在所有员工电脑上吗？
这取决于你的安全需求和资源。但总体趋势是，部署成本正在降低，而收益可能非常显著。
- **传统痛点**：过去部署Sysmon很麻烦，需通过脚本或组策略为每台电脑单独安装，并在海量“噪音”日志中配置复杂规则。
- **关键趋势**：**微软已宣布将在Windows 11和Windows Server 2025中原生集成Sysmon**。这意味着未来它可能变成一个可选组件，一键开启，部署成本将大幅降低。
- **部署建议**：
    - **必须部署**：对**安全要求极高**的服务器（如域控、核心业务服务器）、安全团队自己的工作机、开发测试机。
    - **选择性部署**：普通员工电脑可根据资源决定，优先部署在高风险或高价值岗位。
